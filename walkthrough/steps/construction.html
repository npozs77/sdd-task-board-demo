Construction Phase

Each unit produces: functional design → code → tests → code summary. The functional design is a full specification with numbered pipeline steps, fail branches, TypeScript interfaces, and error matrices.

Every operation is a numbered pipeline with explicit fail branches:

Workflow 1: Create Task

📄 docs/construction/unit-1-data-store/functional-design/business-logic-model.md
title (string), description (string, optional)
    │
    ▼
[1] Trim title string
    │
    ▼
[2] Validate title is non-empty after trim
    │        │
    │     [FAIL] → throw Error("Title must not be empty")
    │
    ▼
[3] Generate task ID via crypto.randomUUID()
    │        │
    │     [FALLBACK] → RFC 4122 v4 manual generation
    │
    ▼
[4] Determine order value: count tasks where column === 'todo'
    │
    ▼
[5] Construct Task object
    │
    ▼
[6] Append to internal tasks array
    │
    ▼
[7] Persist full state to localStorage
    │        │
    │     [FAIL] → console.warn, continue (graceful degradation)
    │
    ▼
[8] Emit 'change' event to all subscribers
    │
    ▼
Return: created Task object

TypeScript interfaces — domain entities

📄 docs/construction/unit-1-data-store/functional-design/business-logic-model.md
interface Task {
  /** UUID v4, immutable after creation. [BR-2] */
  id: string;

  /** Non-empty after trim. [BR-3] */
  title: string;

  /** Any string including empty. Trimmed on save. */
  description: string;

  /** Must be one of VALID_COLUMNS. [BR-1] */
  column: 'todo' | 'in-progress' | 'done';

  /** 0-based position within column. Sequential, no gaps. [FR-6] */
  order: number;
}

Functional Design — excerpt (Unit 3: Controller)

Drag & Drop pipeline

📄 docs/construction/unit-3-controller/functional-design/business-logic-model.md
User releases mouse on .column
    │
    ▼
[4] _onDrop(event):
    │   e.preventDefault()
    │   Find closest .column
    │        │
    │     [FAIL: no column OR no draggedTaskId] → return
    │
    │   Calculate drop position:
    │     cards = column.querySelectorAll('.task-card:not(.dragging)')
    │     FOR i = 0 to cards.length - 1:
    │       midpoint = rect.top + rect.height / 2
    │       IF event.clientY < midpoint:
    │         position = i; BREAK
    │
    │   Call store.moveTask(this._draggedTaskId, targetColumn, position)
    │        │
    │     [FAIL: store throws] → console.warn, continue
    │
    │   Call _cleanup()

Tests — excerpts

Unit tests organized by requirement

📄 tests/store.test.js
describe('title validation [BR-3]', () => {
  it('rejects empty string title', () => {
    expect(() => store.createTask('')).toThrow('Title must not be empty');
  });
  it('rejects whitespace-only title', () => {
    expect(() => store.createTask('   ')).toThrow('Title must not be empty');
  });
  it('accepts title "0" (string-empty, not falsy)', () => {
    expect(store.createTask('0').title).toBe('0');
  });
});

Property-based tests — invariants

📄 tests/store.property.test.js
it('P2: orders within any column are always sequential', () => {
  fc.assert(fc.property(
    fc.array(fc.record({
      action: fc.constantFrom('create', 'move', 'delete', 'reorder'),
      ...
    }), { maxLength: 30 }),
    (operations) => {
      // Apply random operations...
      // INVARIANT: orders are 0, 1, 2, ... (no gaps)
      for (const col of ['todo', 'in-progress', 'done']) {
        const orders = store.getTasks(col).map(t => t.order);
        expect(orders).toEqual(orders.map((_, i) => i));
      }
    }
  ), { numRuns: 100 });
});
