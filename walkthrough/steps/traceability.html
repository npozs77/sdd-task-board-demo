"Why does this code exist? What breaks if I remove it?"

Example: _normalizeOrder()

Function: store.js: _normalizeOrder(column)

Traces to: FR-6 (Task Ordering) — ensures sequential order values after moves/deletes

If removed: Order gaps accumulate → position insertion math breaks → tasks appear in wrong order

Example: event delegation pattern

Pattern: controller.js: single click listener on root

Traces to: NFR-4 (Performance) — event delegation survives re-renders without rebinding

If changed to per-card: Must rebind on every render → performance degrades at scale, risk of memory leaks
