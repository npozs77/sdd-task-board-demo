Functional Requirements — excerpt

Each requirement includes acceptance criteria, step-by-step behaviour, and edge cases:

FR-3: Task Movement Between Columns

📄 docs/inception/requirements/requirements.md

Drag-and-drop flow:

  1. User initiates drag on a task card
  2. Visual feedback: source card becomes semi-transparent, cursor changes
  3. Target column highlights to indicate valid drop zone
  4. User drops the card on a target column
  5. Task moves to target column at the drop position
  6. State is persisted immediately

Drop position calculation:

  • Dropped above existing card → inserts before that card
  • Dropped below all cards → appends to end
  • Dropped on same column → reorder (FR-6)

Constraints: One task at a time. No drag from action buttons.
